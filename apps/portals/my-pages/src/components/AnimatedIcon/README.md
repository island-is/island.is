# Animated Icons

Animated SVG icons used on the My Pages dashboard. Each icon is exported from [SVGator](https://www.svgator.com/), which embeds its animation player directly inside the SVG. The icons play when the user hovers over a dashboard card.

## How it works

There are three pieces to every animated icon:

### 1. SVG file вҖ” `src/assets/icons/sidebar/<name>.svg`

The raw SVG exported from SVGator. It contains the vector artwork **and** the SVGator animation data baked in. Each SVG has a unique root element `id` (e.g. `epyxrSILKQP1` for the election icon).

### 2. React component вҖ” `src/components/AnimatedIcon/<Name>.jsx`

A thin wrapper that:

- Imports the SVG as a React component (`ReactComponent`)
- Runs a `useEffect` on mount that initialises the SVGator player inside the SVG
- Wires up a `click` listener so the animation plays when a click event is dispatched on the SVG element
- Everything that is between `<script><![CDATA[`HERE`]]></script>` should be inside the `useEffect`

```jsx
import { useEffect } from 'react'
import { ReactComponent as Svg } from '../../assets/icons/sidebar/election.svg'

export default function Election() {
  useEffect(() => {
    // SVGator player bootstrap (minified, generated by SVGator export)
    // ...
    const svgEl = document.querySelector('#epyxrSILKQP1')
    if (svgEl?.svgatorPlayer) {
      const setupHover = (player) => {
        svgEl.style.cursor = 'pointer'
        svgEl.addEventListener('click', () => player.restart())
      }
      const p = svgEl.svgatorPlayer
      p.play ? setupHover(p) : p.ready(setupHover)
    }
  }, [])
  return <Svg />
}
```

### 3. ID mapper вҖ” `src/utils/Icons/idMapper.tsx`

Exports two functions used by the Dashboard:

| Function                  | Purpose                                                   |
| ------------------------- | --------------------------------------------------------- |
| `iconTypeToSVG(iconType)` | Returns the React component for an icon type string       |
| `iconIdMapper(iconType)`  | Returns the SVG root element `id` for an icon type string |

The Dashboard uses `iconTypeToSVG` to render the icon inside a `CategoryCard`, and `iconIdMapper` to look up the SVG element in the DOM so it can dispatch a `click` event on hover вҖ” which triggers the SVGator animation.

```tsx
// Dashboard.tsx
const onHover = (id: string) => {
  const el = id && document.getElementById(iconIdMapper(id))
  el && el.dispatchEvent(new Event('click'))
}

// On mouse enter a card:
;<Box onMouseEnter={() => onHover(navRoot.icon?.icon ?? '')}>
  {iconTypeToSVG(navRoot.icon?.icon ?? '')}
</Box>
```

## Adding a new animated icon

Follow these four steps:

### Step 1 вҖ” Design and export from SVGator

1. Create or import your icon design in SVGator and add the animation.
2. Export as **SVG** with the SVGator player embedded (the default export option).
3. Make sure the animation is set to trigger on **click** вҖ” the Dashboard dispatches a `click` event on hover.

### Step 2 вҖ” Add the SVG file

Place the exported SVG in:

```
apps/portals/my-pages/src/assets/icons/sidebar/<iconName>.svg
```

Use camelCase for the filename (e.g. `myNewIcon.svg`). Note the root `id` of the SVG вҖ” you will need it in the next step (it looks like `eXXXXXXXX1`).
Remove the javascript code from `<script><![CDATA[`HERE`]]></script>` (that is moved to the React component instead, see below)

### Step 3 вҖ” Create the React component

Create a new file `src/components/AnimatedIcon/<IconName>.jsx`. Copy the structure from an existing icon (e.g. `Election.jsx`) and:

1. Update the SVG import path to point to your new SVG file.
2. Replace the SVGator player bootstrap code with the one from your SVG export (the minified `!function(...)` block). Everything that is between `<script><![CDATA[`HERE`]]></script>` should be inside the `useEffect`

```jsx
/* eslint-disable */

import { useEffect } from 'react'
import { ReactComponent as Svg } from '../../assets/icons/sidebar/myNewIcon.svg'

export default function MyNewIcon() {
  useEffect(() => {
    // prettier-ignore
    // Paste the minified SVGator bootstrap from your SVG export here
    !function(t,e){ /* ... */ }(this,function(){ /* ... */ });
    ;(function (s, i, o, w, d, a, b) {
      // ... SVGator push call from the SVG export
    })(
      '<player-hash>',
      { root: '<your-svg-root-id>' /* ... */ },
      '__SVGATOR_PLAYER__',
      window,
      document,
    )

    const svgEl = document.querySelector('#<your-svg-root-id>')
    if (svgEl?.svgatorPlayer) {
      const setupHover = (player) => {
        svgEl.style.cursor = 'pointer'
        svgEl.addEventListener('click', () => player.restart())
      }
      const p = svgEl.svgatorPlayer
      p.play ? setupHover(p) : p.ready(setupHover)
    }
  }, [])
  return <Svg />
}
```

### Step 4 вҖ” Register in idMapper

Open `src/utils/Icons/idMapper.tsx` and add your icon to **both** switch statements:

```tsx
// 1. Import at the top
import MyNewIcon from '../../components/AnimatedIcon/MyNewIcon'

// 2. In iconIdMapper вҖ” maps the icon type to the SVG root id
case 'myNewIcon':
  return '<your-svg-root-id>'

// 3. In iconTypeToSVG вҖ” maps the icon type to the React component
case 'myNewIcon':
  return <MyNewIcon />
```

### Extras

- The icon type string (e.g. `'myNewIcon'`) must match the value set in the portal module's navigation definition (`icon.icon` field).
- Also make sure a non animated version is available in the Island UI design library
